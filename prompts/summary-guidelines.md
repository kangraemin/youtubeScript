# 유튜브 주식·경제 transcript 요약 가이드라인

이 파일은 요약 모델이 transcript 1편을 받아 아래 JSON 스키마로 출력하기 위한 톤·스키마 **단일 출처**다. slash command(`.claude/commands/summarize-next.md`)와 분리되어 있어 이 파일만 수정해도 요약 결과 톤이 바뀐다.

**핵심 철학**: "영상 안 봐도 화자들이 한 말을 어느 정도 다 알 수 있는" 수준. 매매·자리·매크로·결론·학습포인트·통계·인용까지 무손실 추출.

## 입력

`scripts/get_next_unsummarized.py`가 stdout JSON으로 메타데이터를 반환하고, transcript 본문은 `/tmp/summarize_target.txt`(병렬 sub-agent는 vid별 고유 경로)에 dump되어 있다. 필드:
- `vid`, `channel`, `channel_slug`, `title`, `published_at`, `url`
- `transcript_path` — `/tmp/summarize_target.txt`
- `transcript_chars`, `transcript_lines`
- `read_chunks`, `chunk_size` — Read 도구로 분할 읽기 위한 정보

## quotes 작성 규칙 (가장 중요)

**`quotes` 필드는 transcript 원문 그대로 옮긴다. 의역·요약·정리 금지.**

**모든 추출 항목**(buys/sells/watchlist/terms/macro_views/chart_levels/verdicts/lessons/data_points/action_items)에 **quotes 1개+ 필수**. **narrative만 예외** (서사 단락이라 quote 없음). quotes가 없으면 그 항목은 빼라.

- 각 항목은 `{"timestamp": "H:MM:SS", "text": "발화 원문"}` 객체
- `timestamp`: transcript 파일 각 라인 앞에 붙어 있는 시각(예: `0:51:33`). 발화 시작 시각으로.
- `text`: transcript 라인을 그대로 복사. 오탈자·말줄임도 그대로. STT 오류로 보이는 표현도 임의 수정 금지(`삼성전자 3전`은 `3전`이 STT 오류여도 그대로 둔다 — 사용자가 검토해야 함)
- 한 항목당 1~3문장. 너무 길면 잘라서 별도 quote로 분리. 너무 짧으면 앞뒤 1~2문장 더 묶기
- 결정적 발언(매매 의사·뷰)이 여러 곳에 흩어져 있으면 `quotes` 배열에 여러 개 넣어도 됨
- ❌ `"quote": "20만500원에 풀매수 했습니다"` ← 의역. 영상에 이렇게 안 나옴
- ✅ `"quotes": [{"timestamp": "0:50:51", "text": "90% 삼성전자 3전에 20만 원대 들어갔죠"}]` ← 발화 그대로

## 추출 규칙

### buys / sells (매수·매도)
- **포함**: "샀습니다", "팔았습니다", "들어갔다", "물타기 했다", "손절했다", "정리했다", "추가 매수했다" 등 **명시적 의사·실행 발언**
- **제외**: "좋아 보인다", "고민 중", "관심 있다" — 이런 건 watchlist로
- 평단·비중을 언급하면 reason에 포함 ("20만500원 풀매수")
- 농담조라도 발화자가 실제 행위로 말한 거면 포함하되 `quotes`로 톤 보존

### watchlist (봐야 할 것)
- "이건 봐야 한다", "주목해야 한다", "사이클이 온다", "곧 강세 올 것" 등 **관전·전망** 발언
- 종목명이 명시되면 ticker 필드도 가능하나, 보통 topic으로 (예: "터보퀀트", "휴머노이드", "코스피 5,000 지지선")

### terms (용어 정리)
- 시청자가 처음 듣거나 이해 어려운 약어·전문어
- 영상 내 설명을 우선 사용. 설명이 없으면 일반 지식으로 보충하되 `source: "보충"` 표시
- 너무 일반적인 용어(주식, 환율 등)는 포함 안 함
- 정의는 1~2문장으로 압축

### macro_views (거시 진단) [신규]
- 환율/금리/원유/채권/지수/CPI/PPI/원자재 등 거시 주제에 대한 화자의 진단·해석
- **5분 이상 다룬 매크로 주제는 무조건 추출**. 영상 핵심 알파인 경우 많음
- `topic`: 한 줄 요약 ("원달러 1400대 돌파"), `view`: 화자의 해석 ("원화 글로벌 선호 약화")
- 단순 시황 전달 (X 지수 Y% 상승)만 있고 해석이 없으면 `data_points`로 보내라

### chart_levels (차트 자리) [신규]
- 종목별 가격대·갭·지지선·저항선 — 화자가 차트를 보고 언급한 모든 종목
- `ticker`: 종목명, `level`: 가격대 한 줄 ("160만원 갭 메꿈 자리"), `reason`: 왜 그 자리인지 ("2단 도약 갭")
- 화자가 차트를 본 종목은 누락 없이 추출 — 영상의 절반이 차트 분석인 경우 많음

### verdicts (조건부 결론·시그널) [신규]
- "X 조건이면 Y한다" 형식 조건부 결론
- `condition`: "다음 주 5% 더 빠지면", `consequence`: "여름 휴가 전까지 장 부재 가능성"
- **영상 후반(70%~끝)의 결론부에서 의무 추출**. 결정적 시점 조언은 매매 룰만큼 중요

### narrative (영상 흐름) [신규]
- 영상 전체 흐름을 시간순 8~15 단계 mini-story로 정리
- 한 영상당 string 1개. 단계 구분은 `→` 또는 줄바꿈
- 시청자가 이걸 읽으면 영상 시간순 골격이 머리에 들어와야 함
- **유일하게 quotes 없는 섹션**

### lessons (학습 포인트) [신규]
- 영상에서 끌어낼 만한 일반화 가능한 통찰. type 4종으로 분류:
  - **rule**: 일반화 가능한 룰 — "주도주 8% 일일 하락 시 매수, 주간 누적 8%는 하락 신호"
  - **counter**: 화자가 반박한 통설 — "통설 X ← 실제 Y" 형식. 시장 컨센서스 대비 화자 견해
  - **reference**: 인용된 책·인물·사건·외부 자료 — "괴테 베르테르 → 베르테르 효과"
  - **analogy**: 비유·메타포 — 머리에 남는 표현. "주식 = 사랑 — 놓아야 다음을 잡음"
- 단순 매매 룰은 rule, 시장 통설을 반박하면 counter, 외부 자료 언급은 reference, 비유는 analogy
- 영상이 학습 가치가 있는 만큼 (강의 영상은 lessons가 많을 수 있고, 시황 영상은 적을 수 있음)

### data_points (통계·숫자) [신규]
- 시청자가 검증·재인용할 수 있는 모든 수치
- 지수 변동, YoY 통계, 외국인 매도 누적, 금리, 거래량, 시총 등
- `datum`: "코스피 -6.12% (5/15)" 같은 단일 사실
- 핵심: **나중에 fact-check하거나 인용할 때 쓸 수 있는 명시적 숫자**만

### action_items (실행 가능) [신규]
- 시청자가 당장 또는 곧 실행할 수 있는 구체적 행동
- "X 종목을 모니터", "Y 자리 오면 매수 고려", "Z 데이터를 확인", "W 시점에 결정"
- 매수·매도(buys/sells)와 다른 점: 자기 매매 발표가 아니라 **시청자 권유**
- 영상 후반·결론부에 자주 나옴

### headline
- 50~80자. 누가/무엇을/왜를 압축
- "라이브 — A 매수 / B 픽 / C 뷰" 같은 슬래시 구조 권장
- 가치 평가(좋다·나쁘다) 단어 자제. 사실 위주.

### raw_summary
- 3~5문장. 영상의 핵심 흐름 + 결정적 발언
- 출연자가 여러 명이면 발언자별로 한 줄씩

## 톤
- 발언자의 표현을 가능한 보존(quotes 활용)
- "유망하다", "추천한다" 같은 단정 평가어 자제 — 발언 그대로
- 영상이 농담·예능 톤이어도 인사이트만 추출, 가십성 발언은 생략

## 길이 정책

**분량 제한 없음.** 영상 길이/내용 밀도에 맞춰 작성한다.

- 2시간 라이브: 15,000~25,000자 (출력 JSON 기준)
- 30분 영상: 5,000~10,000자
- 10분 영상: 2,000~5,000자

**quotes verbatim 무손실이 우선** — 인위적 압축으로 핵심 발화를 잘라내면 안 됨. 항목 수에도 제한 없음.

다만:
- 같은 내용 중복 항목은 quotes 배열에 합쳐라 (별개 항목 X)
- 가십·인사·뉴스 단순 전달 등 인사이트 없는 발화는 생략

## 전수 Read 의무 (긴 영상 대응)

라이브 영상은 1,500~3,000줄(80K~250K자) 규모가 흔하다. Read 도구는 한 번에 25K 토큰 한도(한국어 ~320줄)이라 분할 Read가 필수.

규칙:
- slash command가 `read_chunks` 횟수만큼 offset/limit으로 분할 Read해 끝까지 읽기 전에는 JSON 작성 금지.
- **핵심 매매·종목 발언은 영상 후반(전체의 70%~끝)에 몰려있을 가능성이 높다.** 도입부만 보고 요약하면 quotes에 후반 발언이 빠진다 — 가장 흔한 품질 저하 원인.
- quotes는 도입부·중반·후반 골고루 분포되어야 정상. 한 영역에만 몰려 있으면 누락 의심.
- **verdicts/action_items는 거의 영상 후반에 있음** — 후반 누락 = 결론 누락.

## 검증 후 저장

가이드를 따라 만든 JSON을 `python3 scripts/save_summary.py <vid>` stdin으로 전달한다.
